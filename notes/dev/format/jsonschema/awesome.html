JSON Schema Form 的几种常见情况
  1. 完全 UI 生成
  • 面向数据场景
  • 不会特别好看，但支持的功能多
  1. 提供默认组件联动
  • 面向用户场景
  • 通常会抽象 Field/Form 逻辑
  • 易于扩展
  1. 外置 UI Layout 信息
  • 推荐 - 保持干净的 Schema
  1. 扩展 JSON Schema 内置 UI Layout 信息
  • 扩展后的 JSON Schema 不一定是合法的 JSON Schema
  • 上下关联顺序可能会发生变化
